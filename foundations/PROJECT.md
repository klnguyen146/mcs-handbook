# 1. Project Overview

**Master Computer Science Handbook (MCSH)** is an educational framework designed to help experienced Software Engineers transition into Computer Science, Artificial Intelligence, and academic research.

This project is **not** intended to be another collection of notes, slides, tutorials, or summaries.

Instead, it aims to build a structured **Knowledge System** that transforms an official master's curriculum into a coherent self-learning ecosystem.

The handbook is designed to accompany learners from the beginning of a Master's program until they complete a thesis and become capable of independent research.

---

# 2. Project Vision

Build a complete, structured, and interconnected Computer Science learning system that enables Software Engineers to:

- Understand graduate-level Computer Science.
- Develop strong theoretical foundations.
- Read and understand research papers.
- Conduct independent research.
- Write scientific papers.
- Complete a master's thesis.
- Continue toward PhD-level study if desired.

The long-term goal is to create a learning resource that remains useful for many years rather than serving only as exam preparation.

---

# 3. Mission

The project has five primary missions.

## Mission 1 — Bridge the Knowledge Gap

Help Software Engineers overcome the theoretical gaps between Software Engineering and Computer Science.

Examples include:

- Mathematics
- Algorithms
- Artificial Intelligence
- Machine Learning
- Deep Learning
- Computer Vision
- Natural Language Processing
- Research Methodology

---

## Mission 2 — Build Deep Understanding

Every concept should answer:

- Why does this exist?
- What problem does it solve?
- Why were previous solutions insufficient?
- Where is it applied?
- What are its limitations?
- What research opportunities remain?

The objective is understanding, not memorization.

---

## Mission 3 — Connect Knowledge

Knowledge should never exist in isolation.

Every chapter must be connected to:

- Previous concepts
- Future concepts
- Related courses
- Industrial applications
- Research topics

The learner should gradually build a complete mental model of Computer Science.

---

## Mission 4 — Support Academic Research

The project should prepare learners to:

- Read textbooks
- Read papers
- Review literature
- Design experiments
- Analyze results
- Write scientific reports
- Write a master's thesis

---

## Mission 5 — Long-Term Learning

The handbook should remain useful after graduation.

It should become a long-term reference for:

- Engineers
- Researchers
- Graduate students
- Lifelong learners

---

# 4. Primary Source of Truth

The official Master's curriculum of the University of Information Technology (UIT), Vietnam National University Ho Chi Minh City, is the primary reference for this project.

The curriculum defines:

- Courses
- Learning objectives
- Academic scope

The handbook expands the curriculum by adding prerequisite knowledge required for Software Engineers who may not have studied Computer Science in depth.

The curriculum itself must not be modified.

Instead, additional materials are introduced through Curriculum Mapping.

---

# 5. Target Audience

Primary readers are Software Engineers with approximately 2–5 years of professional experience.

Typical backgrounds include:

- Frontend Development
- Backend Development
- Full-stack Development
- Mobile Development
- DevOps

Readers are expected to possess engineering experience but may lack:

- Mathematics
- Statistics
- Computer Science theory
- Artificial Intelligence
- Machine Learning
- Research experience

The handbook should never assume these topics are already understood.

---

# 6. Scope

The project includes:

- Computer Science foundations
- Mathematics
- Artificial Intelligence
- Machine Learning
- Deep Learning
- Data Engineering
- Knowledge Engineering
- Information Retrieval
- Computer Vision
- Natural Language Processing
- Distributed Systems
- Research Methodology
- Thesis preparation

Each topic is presented according to the official curriculum and extended where necessary.

---

# 7. Out of Scope

The project does **not** focus on:

- Framework tutorials
- Programming language syntax
- Interview preparation
- Competitive programming
- Exam cramming
- Memorization-based learning

Programming examples exist only to reinforce theoretical understanding.

---

# 8. Educational Principles

The project follows these principles.

## Understand Before Memorize

Learners should understand concepts deeply before remembering formulas.

---

## Connect Before Expand

Every new concept should connect to existing knowledge before introducing additional complexity.

---

## Intuition Before Mathematics

Whenever possible:

Problem

↓

Story

↓

Visualization

↓

Intuition

↓

Theory

↓

Mathematics

↓

Algorithm

↓

Implementation

---

## Engineering Before Research

Software Engineering experience should become the foundation for learning Computer Science.

New concepts should be explained using familiar engineering examples whenever appropriate.

---

## Learn by Building

Each major topic should include:

- Exercises
- Mini Projects
- Case Studies
- Reading Suggestions

Learning is reinforced through practical application.

---

# 9. Success Criteria

The project is considered successful if readers can:

- Understand graduate-level courses.
- Read Computer Science textbooks independently.
- Read research papers without relying entirely on secondary explanations.
- Design and implement research projects.
- Write a master's thesis.
- Continue learning independently.

# 11. Guiding Philosophy

This project is built upon one central belief:

> Computer Science is not a collection of isolated subjects.

Instead, it is an interconnected system of ideas.

Understanding those connections is more valuable than memorizing individual techniques.

# 14. Guiding Principle

This principle overrides all implementation decisions.
